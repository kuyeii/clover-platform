# 首页布局重构说明

## 原始布局问题
- 顶部完整 topbar 和左上角品牌区占据了过多注意力。
- 左侧 sidebar 让首页更像后台控制台，而不是统一入口工作台。
- 首页说明文字和技术信息过多，削弱了四个业务模块的视觉中心。
- “总览 / 模块 / 工作台” 同时存在，导航语义重复。

## 当前重构方向
- 首页改为工作台页面。
- 页面顶部只保留一个主标题。
- 标题下方使用居中的浮动导航条。
- 导航只保留：工作台、知识库、设置。
- 用户占位按钮与头像并入导航条右侧。
- 页面主体用四叶草模块区作为唯一主视觉中心。

## 为什么移除 topbar 和 sidebar
- 统一入口门户的核心任务是分发业务模块，而不是承载复杂后台操作。
- topbar 和 sidebar 会分散用户视线，降低首页聚焦度。
- 居中导航条更适合轻量入口门户，也更符合当前极简工作台设计。

## 为什么把“总览”改为“工作台”
- 原有“总览”与“工作台”表达的都是门户默认页能力。
- 保留两个入口会制造重复概念。
- 统一改为“工作台”后，语义更稳定，也便于后续接入个人视角和最近使用能力。

## 知识库页面定位
- 当前知识库页只表现为平台内已上传数据的静态占位列表。
- 不展示外部接入状态。
- 不强调外部工作流平台。
- 不接真实上传接口和真实后端请求。
